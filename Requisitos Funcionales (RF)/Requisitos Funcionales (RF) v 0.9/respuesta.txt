### Sobre la lógica de periodicidad (lo más importante)

Tu descripción del ciclo es **exactamente como funcionan los CMMS profesionales**. Déjame dibujarlo:

```
┌─────────────────────────────────────────────────────┐
│  CICLO DE MANTENIMIENTO PREVENTIVO                  │
│                                                     │
│  1. Administrador programa MP                       │
│     → Crea OT preventiva (🟢 VERDE en calendario)   │
│                                                     │
│  2. Técnico completa la OT                          │
│     → ultima_fecha = fecha de completado            │
│     → Sistema sugiere próxima fecha (recordatorio)  │
│     → Recordatorio (🟡 AMARILLO en calendario)      │
│                                                     │
│  3. Administrador convierte recordatorio en OT      │
│     → Nueva OT preventiva (🟢 VERDE)                │
│     → Vuelve al paso 2                              │
│                                                     │
│  4. Si la fecha del recordatorio pasa sin acción    │
│     → Recordatorio vencido (🔴 ROJO en calendario)  │
│     → Sigue siendo recordatorio, solo cambia color  │
│     → Se puede convertir en OT cuando se quiera     │
└─────────────────────────────────────────────────────┘
```

**Esto es brillante porque**:
- El `frecuencia_dias` (ej: 90) se usa SOLO para sugerir la próxima fecha
- El `proxima_fecha` es la fecha REAL que aparece en el calendario
- El color depende del estado: verde (OT activa), amarillo (recordatorio), rojo (vencido)
- No necesitas campos complejos como `condiciones_activacion` o `logica_periodicidad` (esos son para v1.0+)
